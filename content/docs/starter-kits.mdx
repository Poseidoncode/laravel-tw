<a name="introduction"></a>
## 簡介 (Introduction)

為了讓您在建立新的 Laravel 應用程式時能有個領先的起跑點，我們很高興能提供 [應用程式啟動套件 (application starter kits)](https://laravel.com/starter-kits)。這些啟動套件能讓您在建立下一個 Laravel 應用程式時快速上手，並包含了註冊和驗證應用程式使用者所需的路由、控制器和視圖。這些啟動套件使用 [Laravel Fortify](/docs/fortify) 來提供驗證功能。

雖然我們歡迎您使用這些啟動套件，但它們並非強制性的。您完全可以透過安裝一份全新的 Laravel，從頭開始建立您自己的應用程式。無論哪種方式，我們都知道您將會打造出色的作品！

<a name="creating-an-application"></a>
## 使用啟動套件建立應用程式 (Creating An Application)

要使用我們的啟動套件之一來建立新的 Laravel 應用程式，您首先應該 [安裝 PHP 和 Laravel CLI 工具](/docs/installation#installing-php)。如果您已經安裝了 PHP 和 Composer，您可以透過 Composer 安裝 Laravel 安裝程式 CLI 工具：

```shell
composer global require laravel/installer
```

接著，使用 Laravel 安裝程式 CLI 建立一個新的 Laravel 應用程式。Laravel 安裝程式會提示您選擇偏好的啟動套件：

```shell
laravel new my-app
```

建立 Laravel 應用程式後，您只需要透過 NPM 安裝其前端依賴項目並啟動 Laravel 開發伺服器：

```shell
cd my-app
npm install && npm run build
composer run dev
```

一旦您啟動了 Laravel 開發伺服器，您的應用程式就可以在網頁瀏覽器中透過 [http://localhost:8000](http://localhost:8000) 存取。

<a name="available-starter-kits"></a>
## 可用的啟動套件 (Available Starter Kits)

<a name="react"></a>
### React

我們的 React 啟動套件提供了一個強大且現代化的起點，用於使用 [Inertia](https://inertiajs.com) 建立具有 React 前端的 Laravel 應用程式。

Inertia 允許您使用經典的伺服器端路由和控制器來建立現代化的單頁 React 應用程式。這讓您可以享受 React 的前端強大功能，同時結合 Laravel 令人難以置信的後端生產力和極速的 Vite 編譯。

React 啟動套件使用了 React 19、TypeScript、Tailwind 和 [shadcn/ui](https://ui.shadcn.com) 元件庫。

<a name="vue"></a>
### Vue

我們的 Vue 啟動套件提供了一個絕佳的起點，用於使用 [Inertia](https://inertiajs.com) 建立具有 Vue 前端的 Laravel 應用程式。

Inertia 允許您使用經典的伺服器端路由和控制器來建立現代化的單頁 Vue 應用程式。這讓您可以享受 Vue 的前端強大功能，同時結合 Laravel 令人難以置信的後端生產力和極速的 Vite 編譯。

Vue 啟動套件使用了 Vue Composition API、TypeScript、Tailwind 和 [shadcn-vue](https://www.shadcn-vue.com/) 元件庫。

<a name="livewire"></a>
### Livewire

我們的 Livewire 啟動套件為使用 [Laravel Livewire](https://livewire.laravel.com) 前端建立 Laravel 應用程式提供了完美的起點。

Livewire 是一種僅使用 PHP 即可建立動態、反應式前端 UI 的強大方式。對於主要使用 Blade 樣板且正在尋找比 React 和 Vue 等 JavaScript 驅動的 SPA 框架更簡單替代方案的團隊來說，這是一個絕佳的選擇。

Livewire 啟動套件使用了 Livewire、Tailwind 和 [Flux UI](https://fluxui.dev) 元件庫。

<a name="starter-kit-customization"></a>
## 啟動套件自訂 (Starter Kit Customization)

<a name="react-customization"></a>
### React

我們的 React 啟動套件是使用 Inertia 2、React 19、Tailwind 4 和 [shadcn/ui](https://ui.shadcn.com) 建構的。與我們所有的啟動套件一樣，所有的後端和前端程式碼都存在於您的應用程式中，允許進行完全的自訂。

大部分的前端程式碼位於 `resources/js` 目錄中。您可以自由修改任何程式碼以自訂應用程式的外觀和行為：

```text
resources/js/
├── components/    # 可重複使用的 React 元件
├── hooks/         # React hooks
├── layouts/       # 應用程式版面配置
├── lib/           # 工具函式和設定
├── pages/         # 頁面元件
└── types/         # TypeScript 定義
```

要發佈額外的 shadcn 元件，請先 [找到您想要發佈的元件](https://ui.shadcn.com)。然後，使用 `npx` 發佈該元件：

```shell
npx shadcn@latest add switch
```

在這個範例中，該指令會將 Switch 元件發佈到 `resources/js/components/ui/switch.tsx`。一旦元件被發佈，您就可以在您的任何頁面中使用它：

```jsx
import { Switch } from "@/components/ui/switch";

const MyPage = () => {
  return (
    <div>
      <Switch />
    </div>
  );
};

export default MyPage;
```

<a name="react-available-layouts"></a>
#### 可用的版面配置 (Available Layouts)

React 啟動套件包含兩種不同的主要版面配置供您選擇：「側邊欄 (sidebar)」版面配置和「標頭 (header)」版面配置。側邊欄版面配置是預設的，但您可以透過修改應用程式 `resources/js/layouts/app-layout.tsx` 檔案頂部匯入的版面配置來切換到標頭版面配置：

```js
import AppLayoutTemplate from "@/layouts/app/app-sidebar-layout"; // [tl! remove]
import AppLayoutTemplate from "@/layouts/app/app-header-layout"; // [tl! add]
```

<a name="react-sidebar-variants"></a>
#### 側邊欄變體 (Sidebar Variants)

側邊欄版面配置包含三種不同的變體：預設側邊欄變體、「內嵌 (inset)」變體和「浮動 (floating)」變體。您可以透過修改 `resources/js/components/app-sidebar.tsx` 元件來選擇您最喜歡的變體：

```text
<Sidebar collapsible="icon" variant="sidebar"> [tl! remove]
<Sidebar collapsible="icon" variant="inset"> [tl! add]
```

<a name="react-authentication-page-layout-variants"></a>
#### 驗證頁面版面配置變體 (Authentication Page Layout Variants)

React 啟動套件中包含的驗證頁面（例如登入頁面和註冊頁面）也提供了三種不同的版面配置變體：「簡單 (simple)」、「卡片 (card)」和「分割 (split)」。

要更改您的驗證版面配置，請修改應用程式 `resources/js/layouts/auth-layout.tsx` 檔案頂部匯入的版面配置：

```js
import AuthLayoutTemplate from "@/layouts/auth/auth-simple-layout"; // [tl! remove]
import AuthLayoutTemplate from "@/layouts/auth/auth-split-layout"; // [tl! add]
```

我們的 Livewire 啟動套件是使用 Livewire 3、Tailwind 和 [Flux UI](https://fluxui.dev/) 建構的。與我們所有的啟動套件一樣，所有的後端和前端程式碼都存在於您的應用程式中，允許進行完全的自訂。

```text

```

#### 傳統 Livewire 元件

前端程式碼位於 `resources/views` 目錄中，而 `app/Livewire` 目錄包含 Livewire 元件對應的後端邏輯。

<a name="two-factor-authentication"></a>
## 兩因素驗證 (Two Factor Authentication)

所有啟動套件都包含由 [Laravel Fortify](/docs/fortify#two-factor-authentication) 驅動的內建兩因素驗證 (2FA)，為使用者帳戶增加了一層額外的安全性。使用者可以使用任何支援基於時間的一次性密碼 (TOTP) 的驗證器應用程式來保護他們的帳戶。

兩因素驗證預設為啟用，並支援 [Fortify](/docs/fortify#two-factor-authentication) 提供的所有選項：

```php
Features::twoFactorAuthentication([
    'confirm' => true,
    'confirmPassword' => true,
]);
```

<a name="workos"></a>
## WorkOS AuthKit 認證

預設情況下，React、Vue 和 Livewire 啟動套件都使用 Laravel 內建的驗證系統來提供登入、註冊、密碼重設、電子郵件驗證等功能。此外，我們也提供每個啟動套件的 [WorkOS AuthKit](https://authkit.com) 驅動變體，提供：

<div className="content-list" markdown="1">

- 社群驗證 (Google, Microsoft, GitHub, and Apple)
- Passkey 驗證
- 基於電子郵件的「Magic Auth」
- SSO (單一登入)

</div>

使用 WorkOS 作為您的驗證提供者 [需要一個 WorkOS 帳戶](https://workos.com)。WorkOS 為每月活躍使用者不超過 100 萬的應用程式提供免費驗證。

要使用 WorkOS AuthKit 作為您應用程式的驗證提供者，請在透過 `laravel new` 建立新的啟動套件驅動應用程式時選擇 WorkOS 選項。

### 設定您的 WorkOS 啟動套件

使用 WorkOS 驅動的啟動套件建立新應用程式後，您應該在應用程式的 `.env` 檔案中設定 `WORKOS_CLIENT_ID`、`WORKOS_API_KEY` 和 `WORKOS_REDIRECT_URL` 環境變數。這些變數應與您在 WorkOS 儀表板中為您的應用程式提供的值相符：

```ini
WORKOS_CLIENT_ID=your-client-id
WORKOS_API_KEY=your-api-key
WORKOS_REDIRECT_URL="${APP_URL}/authenticate"
```

此外，您應該在 WorkOS 儀表板中設定應用程式首頁 URL。此 URL 是使用者登出您的應用程式後將被重新導向的地方。

<a name="configuring-authkit-authentication-methods"></a>
#### 設定 AuthKit 驗證方法 (Configuring Authkit Authentication Methods)

使用 WorkOS 驅動的啟動套件時，我們建議您在應用程式的 WorkOS AuthKit 設定中停用「電子郵件 + 密碼」驗證，僅允許使用者透過社群驗證提供者、Passkeys、「Magic Auth」和 SSO 進行驗證。這讓您的應用程式可以完全避免處理使用者密碼。

<a name="configuring-authkit-session-timeouts"></a>
#### 設定 AuthKit 工作階段逾時 (Configuring Authkit Session Timeouts)

此外，我們建議您設定 WorkOS AuthKit 工作階段閒置逾時，以符合您 Laravel 應用程式設定的工作階段逾時閾值，通常為兩小時。

<a name="inertia-ssr"></a>
### Inertia SSR

React 和 Vue 啟動套件與 Inertia 的 [伺服器端渲染 (SSR)](https://inertiajs.com/server-side-rendering) 功能相容。要為您的應用程式建構與 Inertia SSR 相容的 bundle，請執行 `build:ssr` 指令：

```shell
npm run build:ssr
```

為了方便起見，也提供了一個 `composer dev:ssr` 指令。此指令會在為您的應用程式建構與 SSR 相容的 bundle 後，啟動 Laravel 開發伺服器和 Inertia SSR 伺服器，讓您可以使用 Inertia 的伺服器端渲染引擎在本地測試您的應用程式：

```shell
composer dev:ssr
```

laravel new my-app --using=example/starter-kit

<a name="creating-starter-kits"></a>
#### 建立啟動套件 (Creating Starter Kits)

為了確保其他人可以使用您的啟動套件，您需要將其發佈到 [Packagist](https://packagist.org)。您的啟動套件應該在其 `.env.example` 檔案中定義其所需的環境變數，任何必要的安裝後指令都應該列在啟動套件 `composer.json` 檔案的 `post-create-project-cmd` 陣列中。

<a name="faqs"></a>
### 常見問題 (Faqs)

<a name="faq-upgrade"></a>
#### 如何升級？ (Faq Upgrade)

每個啟動套件都為您的下一個應用程式提供了一個堅實的起點。擁有程式碼的完全所有權，您可以完全按照您的構想進行調整、自訂和建構應用程式。但是，不需要更新啟動套件本身。

<a name="faq-enable-email-verification"></a>
#### 如何啟用電子郵件驗證？ (Faq Enable Email Verification)

可以透過取消註解 `App/Models/User.php` 模型中的 `MustVerifyEmail` 匯入，並確保模型實作 `MustVerifyEmail` 介面來新增電子郵件驗證：

```php
<?php

namespace App\Models;

use Illuminate\Contracts\Auth\MustVerifyEmail;
// ...

class User extends Authenticatable implements MustVerifyEmail
{
    // ...
}
```

註冊後，使用者將收到一封驗證電子郵件。要限制在使用者電子郵件地址通過驗證之前存取某些路由，請將 `verified` 中介軟體新增到路由中：

```php
Route::middleware(['auth', 'verified'])->group(function () {
    Route::get('dashboard', function () {
        return Inertia::render('dashboard');
    })->name('dashboard');
});
```

> [!NOTE]
> 使用 [WorkOS](#workos) 變體的啟動套件時，不需要電子郵件驗證。

<a name="faq-modify-email-template"></a>
#### 如何修改預設的電子郵件樣板？ (Faq Modify Email Template)

您可能想要自訂預設的電子郵件樣板，以更符合您應用程式的品牌形象。要修改此樣板，您應該使用以下指令將電子郵件視圖發佈到您的應用程式：

```
php artisan vendor:publish --tag=laravel-mail
```

這將在 `resources/views/vendor/mail` 中產生數個檔案。您可以修改這些檔案中的任何一個，以及 `resources/views/vendor/mail/themes/default.css` 檔案，以更改預設電子郵件樣板的外觀和感覺。
